# boilerplate-go

生产级 Go 单仓库多服务项目模板：清晰目录规范 + 完整工程实践示例。

单仓库多服务让所有服务的代码集中在一处，既降低多库维护成本，又便于统一规范。在 AI 时代，这一模式价值更突出：**AI 能直接看到完整代码上下文**，无需跨库跳转，避免「跨库理解、跨服务推理」时的上下文断裂，使 AI 辅助编程与代码理解更高效。

## 🎯 适用场景

✅ **推荐使用**：

- 小团队或个人微服务项目，希望减少多仓库的维护与协作成本
- 需要统一代码规范、依赖版本的团队项目
- 希望借助 AI 做全链路理解与改动的项目（单仓库便于 AI 获得完整上下文）
